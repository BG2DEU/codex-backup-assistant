# 安全与隐私反馈

## 不要公开提交敏感数据

请不要在 GitHub Issue、Pull Request、讨论区或其他公开位置上传：

- 真实备份包或私人项目源码。
- 完整日志、完整恢复报告或未脱敏截图。
- 用户名、电脑名、完整本机路径或私人项目名称。
- Git 仓库私有地址、客户资料或商业数据。
- 密码、Cookie、登录令牌、API Key、私钥、证书或 `.env` 内容。

普通错误反馈请使用
[志愿测试反馈表](https://github.com/BG2DEU/codex-backup-assistant/issues/new?template=test-report.yml)，
并只提供经过脱敏的最小必要信息。

## 安全问题

如果问题可能导致凭据泄露、任意文件覆盖、目录穿越、备份包篡改未被发现或其他安全影响，请不要直接公开技术细节或真实样本。

当前项目尚未建立专用安全邮箱。请先创建一个不包含利用细节的 GitHub Issue，标题使用
`[安全问题，请求私下联系]`，只说明受影响版本和问题类型，等待维护者提供后续联系方法。

## 当前安全边界

- 软件默认排除登录令牌、浏览器会话、机器标识和 sandbox secrets。
- 备份包可能包含私人源码、对话、配置和路径，应只保存在可信介质中。
- 当前预览版没有对整个备份包进行加密。
- 当前发布程序尚未进行代码签名，请只从本仓库 Releases 下载并校验 SHA-256。
