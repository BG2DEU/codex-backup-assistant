# Codex 换机助手用户操作脚本

## 适用前提

- 旧电脑可以正常打开本工具。
- 新电脑已经安装 Codex，并且已经至少启动、登录过一次 Codex。
- 新电脑恢复前必须完全退出 Codex。
- 移动硬盘或 U 盘空间足够，并且只由可信的人保管。

## 旧电脑导出

1. 插入移动硬盘。
2. 打开桌面上的 `Codex换机助手-预览版.exe`。
3. 点击“开始扫描”。
4. 检查项目和 Codex 数据列表。
5. 点击“生成备份计划（只读）”。
6. 如果提示 Codex 仍在运行，请退出 Codex 后重新扫描。
7. 点击“导出备份包”。
8. 选择移动硬盘中的目标文件夹。
9. 等待导出完成。
10. 导出成功后，移动硬盘中会出现一个完整备份包目录，里面包含：
    - `manifest.json`
    - `package-index.json`
    - `payload`
    - `portable`
    - `tools/Codex换机助手.exe`
    - `tools/新电脑恢复说明.txt`

## 新电脑恢复

1. 在新电脑上确认 Codex 已经安装并登录过。
2. 完全退出 Codex。
3. 插入移动硬盘。
4. 打开备份包里的 `tools/Codex换机助手.exe`，或打开桌面预览版。
5. 点击“查看新电脑操作说明”，确认前提满足。
6. 点击“校验备份包”。
7. 选择包含 `manifest.json` 的备份包根目录。
8. 校验通过后，点击“选择备份并恢复”。
9. 再次选择备份包根目录。
10. 选择项目恢复目录，例如 `文档/Codex恢复项目`。
11. 当软件询问是否恢复 Codex 原生状态时，默认选择“否”。
12. 确认恢复。
13. 恢复完成后查看底部摘要。
14. 点击“打开恢复报告”。

## 成功判断

- 状态栏显示“恢复完成”。
- 项目出现在你选择的项目恢复目录中。
- 如果新电脑已有同名项目，旧电脑项目会另存为 `项目名-从备份恢复-*`。
- 通用对话位于 `Codex换机恢复资料/Codex通用对话`。
- HTML 恢复报告可以打开，并显示恢复项、跳过项、问题和回滚目录。

## 常见情况

### 备份包不完整

重新回到旧电脑导出，不要使用带 `.incomplete` 后缀或包含 `INCOMPLETE.json` 的目录恢复。

### 选错目录

校验或恢复时请选择包含 `manifest.json` 和 `package-index.json` 的备份包根目录，不要选择 `payload`、`portable` 或 `tools` 子目录。

### Codex 没有完全退出

如果尝试恢复 Codex 原生状态，必须完全退出 Codex。推荐默认不恢复原生状态，先恢复项目、通用对话和安全配置。

### 新电脑已有同名项目

默认不会覆盖。工具会另存为 `项目名-从备份恢复-*`。

## 不会迁移的内容

- 旧电脑登录令牌。
- 浏览器会话。
- 机器标识。
- sandbox secrets。
- 可重建缓存和临时运行文件。

## 失败后如何排查

1. 查看软件底部状态栏里的故障代码或提示。
2. 如果有恢复报告，点击“打开恢复报告”。
3. 点击“打开日志目录”，找到当天的 `codex-backup-YYYYMMDD.log`。
4. 对照 [故障代码与日志说明](./故障代码与日志说明.md) 判断原因。
5. 不要删除备份包和回滚目录，直到确认恢复结果正确。
