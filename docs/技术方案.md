## 2. 技术选型

### 2.1 正式产品

正式产品采用 C#、当时受支持的 .NET LTS 版本和 WPF。发布时使用 `win-x64` 自包含产物，使新电脑不需要预先安装 .NET 运行时。

选择原因：

- 产品定位为 Windows 专用工具，WPF 对长路径、物理磁盘、进程、文件权限和系统对话框的集成直接。
- .NET 内置流式 I/O、加密原语、JSON、取消标记和异步任务，能减少核心备份引擎的第三方依赖。
- 自包含发布能满足“移动硬盘上携带一个可直接运行的恢复程序”的需求。
- 相比 Electron，产物体积和运行时开销更容易控制；相比纯 PowerShell，更适合长期维护、强类型数据模型和系统化测试。

阶段 B 已通过 Windows 系统软件源安装 .NET SDK 10.0.301，并使用 `global.json` 锁定同一特性带的最新补丁版本。最终发布仍使用自包含模式，新电脑不需要手动安装 .NET SDK 或运行时。

### 2.2 阶段 A 探针

发现探针使用 Windows PowerShell 5.1，原因是 Windows 10/11 通常内置，且阶段 A 只需验证数据发现和风险分类。探针不是正式备份引擎，不会演变为依赖大量脚本拼接的产品架构。

## 3. 系统分层

```text
CodexBackup.App.Wpf
├─ Presentation        向导界面、状态、进度、用户确认
├─ Application         扫描、导出、校验、导入用例编排
├─ Domain              备份计划、数据分类、恢复等级、结果模型
├─ Infrastructure      Windows 文件系统、磁盘、进程、Git 和加密实现
├─ Adapters.Codex      按 Codex 结构版本发现、导出和恢复
└─ Reporting           manifest、JSON/HTML 报告和脱敏输出
```

依赖方向必须指向 Domain，Codex 内部路径、数据库名和版本探测不得渗透进界面或通用复制引擎。

## 4. 核心数据模型

### 4.1 DiscoveryReport

- `schemaVersion`：发现报告版本。
- `generatedAtUtc`：生成时间。
- `host`：非凭据类主机信息。
- `storage`：盘符、物理磁盘标识、文件系统和可用空间。
- `codex`：Codex 根目录、数据分类、文件数、字节数和默认策略。
- `projects`：项目路径、发现原因、Git 摘要、文件规模和风险。
- `warnings`：权限、锁文件、遍历失败和不支持项。

### 4.2 项目发现管线

项目发现采用“Codex 记录为主、项目标志补漏、用户最终确认”的管线，不进行无差别的整盘文件内容扫描。

1. **主来源**：从 Codex 会话 JSONL、会话索引和可识别状态中提取 `cwd` 或等价工作路径。
2. **根目录解析**：从每个有效工作路径向上查找 Git 和常见项目标志，将会话子目录归并到真正项目根。
3. **轻量补漏**：在用户目录、已确认项目根和挂载固定磁盘上遍历目录元数据，仅查找 `.git` 及项目标志，跳过系统目录和依赖缓存。
4. **归并去重**：同一项目可同时具有 `CodexSessionPath`、`GitRepository`、`ProjectMarker` 和 `UserAdded` 多个来源。
5. **用户确认**：界面默认选中高信心项目，对丢失路径、非 Git 目录和超大目录进行单独提示。

阶段 A 验证表明：Codex 会话中的 `cwd` 是高价值索引，但不能被当作完整项目注册表；仍需要项目标志补漏和用户最终确认。

### 4.3 数据政策

每个 Codex 顶层项都必须归入以下策略之一：

- `Include`：主要数据，换机导出默认包含。
- `IncludePortableAndNative`：同时保存原始数据和通用导出。
- `InventoryOnly`：仅记录清单，目标电脑重建。
- `ExcludeCredential`：凭据或机器绑定数据，默认排除。
- `ExcludeVolatile`：缓存、临时文件或运行时数据。
- `UnknownReviewRequired`：新版本未知项，必须进入人工确认，不得静默排除。

## 5. 只读约束

1. 发现探针只允许写入用户明确传入的 `OutputPath`。
2. 探针不读取项目文件内容，只读取文件系统元数据和 Git 命令输出。
3. Git 命令必须在 `GIT_OPTIONAL_LOCKS=0` 环境下运行，避免可选索引刷新。
4. 遇到符号链接或目录联接时只记录而不跟随。
5. 权限不足、路径失效和读取失败必须变成警告，不得使整个扫描无报告崩溃。
6. 探针不会解析或输出 `auth.json`、`.env`、私钥或令牌内容。

## 6. 阶段 A 出口条件

- 可在测试环境识别 Codex 数据根目录及顶层分类。
- 可从 Codex 会话元数据提取工作路径，并区分直接命中项目和补漏项目。
- 可识别用户文档、桌面和已确认根目录中的 Git 项目及常见非 Git 项目。
- 可在不改变 Git 工作树的前提下记录分支、变更数和远程有无。
- 可识别同物理磁盘上的不同盘符。
- 报告不包含文件内容、密码、完整令牌或私钥。
- 探针烟雾测试和真实机器扫描都必须通过。
