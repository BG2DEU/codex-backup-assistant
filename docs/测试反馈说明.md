# 测试反馈说明

## 项目背景

本项目是 Codex 换机助手的 `1.0.0-preview.1` 测试版，目标是帮助 Windows 用户把旧电脑上的 Codex 相关项目、可读对话、配置说明和恢复工具导出到移动硬盘，并在新电脑上校验后恢复。

项目由作者提出需求和使用场景，并在作者没有软件编程背景的情况下，通过 Codex 协助完成需求文档、技术方案、代码实现、自动测试、桌面程序和发布包。作者目前没有第二台新电脑完成真实换机测试，因此需要其他用户协助验证。

## 希望测试者验证什么

建议优先测试以下场景：

1. 在旧电脑或测试电脑上启动桌面程序。
2. 点击扫描，确认程序是否能发现 Codex 数据和相关项目。
3. 导出到一个临时文件夹、移动硬盘或 U 盘。
4. 导出完成后确认是否生成备份包、校验信息和报告。
5. 在另一台电脑、虚拟机或新 Windows 用户环境中打开备份包内的恢复工具。
6. 执行备份包校验。
7. 在不覆盖重要数据的前提下测试恢复。
8. 查看恢复报告和日志，确认失败时是否能定位原因。

如果没有第二台电脑，也可以只测试旧电脑导出、备份包校验和报告生成。

## 重要安全提醒

- 不要用唯一的重要项目做第一次恢复测试。
- 不要把真实备份包公开上传到 GitHub。
- 不要公开上传完整日志，日志可能包含本机路径、项目名或用户名。
- 不要公开上传包含账号、令牌、私有仓库地址、客户资料或商业代码的截图。
- 建议先使用临时项目、测试项目或可公开项目进行验证。

## 反馈时请提供

请尽量提供以下信息：

- Windows 版本，例如 Windows 10 或 Windows 11。
- Codex 是否已安装、是否已启动并登录过一次。
- 测试的是导出、校验、恢复，还是完整换机流程。
- 程序版本：`1.0.0-preview.1`。
- 是否使用移动硬盘、U 盘、本机文件夹或网盘同步目录。
- 程序界面显示的错误代码或错误摘要。
- 恢复报告中的失败项目摘要。
- 日志中与错误相关的少量片段。

- 会使用 GitHub 的测试者，可以通过仓库的测试反馈表提交。

## 当前已知限制

- 不迁移旧电脑登录令牌、浏览器会话、机器标识或 sandbox secrets。
- 新电脑需要已经安装 Codex，并至少启动和登录过一次。
- 恢复前需要完全退出 Codex。
- Codex 原生状态恢复属于高风险选项，默认关闭。
- 首版仍需要真实新电脑、虚拟机和不同 Windows 环境的验证。
