## 已完成

### C-01 恢复领域模型

- 新增 `RestoreRequest`、`RestorePlan`、`RestorePlanItem`、`RestoreConflictPolicy`、`RestoreProgress` 和 `RestoreResult`。
- 恢复请求明确区分备份包、项目恢复根目录、Codex 目标目录和通用恢复资料目录。
- 项目默认使用“另存”策略，避免覆盖新电脑已有项目。
- Codex 配置默认使用“合并并保留现有文件”策略。
- Codex 原生状态默认不恢复，必须由用户明确确认。

### C-02 恢复计划与安全边界

- `RestorePlanBuilder` 会读取 `package-index.json`，把备份项映射到新电脑目标路径。
- 登录凭据、机器绑定标识、sandbox secrets 和 Chrome native host 状态永不恢复。
- 备份包适配器版本不兼容时，阻止 Codex 原生状态恢复。
- 备份包内自带的恢复工具只作为工具使用，不作为用户数据恢复。
- 重复目标路径、危险相对路径和无可恢复项会变成阻止问题。

### C-03 恢复执行与回滚

- `BackupRestoreEngine` 在写入前先完整校验备份包。
- 恢复后逐文件重新计算 SHA-256，确认目标文件与备份 manifest 一致。
- 覆盖类恢复先建立回滚副本，失败或取消时按回滚记录清理或恢复。
- 恢复完成后生成 JSON 与 HTML 恢复报告。
- 如果用户尝试恢复 Codex 原生状态，会检查新电脑 Codex 目录是否存在、Codex 是否已完全退出、数据库是否未被锁定。

### C-04 备份包自带恢复工具

- 每个完成备份包会加入：
  - `tools/Codex换机助手.exe`
  - `tools/新电脑恢复说明.txt`
- 新电脑可直接从备份包内启动恢复工具，再选择该备份包执行校验或恢复。

### C-05 桌面恢复入口

- “检查现有备份”按钮已接入 `BackupPackageVerifier`。
- “在新电脑恢复”按钮已接入 `BackupRestoreEngine`。
- 恢复流程会依次选择备份包、选择项目恢复根目录、询问是否尝试 Codex 原生状态恢复，并进行最终确认。
- 恢复中显示阶段、文件数、字节数、速度和预计剩余时间。
- 提供恢复取消按钮，取消后执行安全回滚。

### C-06 桌面独立预览版发布

- 已重新发布 self-contained 单文件预览版。
- 文件大小和 SHA-256 以当前发布目录中的 `RELEASE-MANIFEST.txt` 和 `SHA256.txt` 为准。
- 启动烟雾测试通过，进程正常运行，窗口标题为“Codex 换机助手 1.0.0-preview.1”。

### C-07 桌面命令层换机演练

- 新增 `CodexBackup.App.Tests`，覆盖桌面按钮背后的 ViewModel 命令流程。
- `VerifyBackupCommand` 测试会创建临时备份包，再通过模拟用户选择执行备份包校验。
- `RestoreCommand` 测试会创建临时旧电脑项目和 Codex 会话，导出备份包，再模拟新电脑恢复。
- 恢复演练验证：
  - 已有同名项目会被保留。
  - 旧电脑项目会另存为 `Demo-从备份恢复-*`。
  - 通用对话会恢复到 `Codex换机恢复资料/Codex通用对话`。
  - 状态栏返回“恢复完成”。
- `AsyncRelayCommand` 增加可等待的 `ExecuteAsync`，用于自动测试可靠等待异步按钮流程结束；界面 `ICommand` 用法不变。

### C-08 恢复结果展示优化

- `RestoreResult` 新增 `ReportJsonPath` 和 `ReportHtmlPath`，恢复引擎会把实际生成的报告路径返回给桌面界面。
- 桌面底部新增恢复结果面板，恢复完成后显示：
  - 恢复项目数量。
  - 恢复 Codex 数据数量。
  - 通用对话是否恢复。
  - 同名项目另存数量。
  - 跳过项、失败项和首个问题提示。
  - HTML 恢复报告路径。
- 新增“打开恢复报告”按钮，可直接打开最近一次恢复生成的 HTML 报告。
- 自动测试已覆盖恢复报告路径返回、恢复摘要展示和打开报告按钮可用状态。

### C-09 新电脑操作说明

- “在新电脑恢复”卡片明确显示前提：新电脑已经安装并登录过 Codex，恢复前必须完全退出 Codex。
- 新增“查看新电脑操作说明”按钮。
- 操作说明不再把“安装 Codex”作为恢复流程的一步，而是要求用户先完成新电脑自己的 Codex 初始化：
  - 已安装 Codex。
  - 已至少启动并登录过一次 Codex。
  - 恢复前完全退出 Codex。
  - 插入移动硬盘并打开备份包内的恢复工具或桌面预览版。
  - 先校验备份包，再执行恢复。
  - 默认不恢复 Codex 原生状态。
- 自动测试确认说明入口使用“已经安装并登录过 Codex”的前提，不引导用户在恢复流程中重新安装 Codex。

### C-10 备份包内说明与恢复报告增强

- `tools/新电脑恢复说明.txt` 已与桌面说明同步：
  - 新电脑已经安装 Codex。
  - 已经至少启动并登录过一次 Codex。
  - 恢复前请完全退出 Codex。
  - 默认选择不恢复 Codex 原生状态。
- HTML 恢复报告增强为可读表格，包含：
  - 备份 ID、备份包位置、完成时间和回滚目录。
  - 恢复文件数、恢复体量、恢复项、跳过项和失败/回滚项。
  - 每个恢复项的名称、类型、状态、目标位置、冲突策略和说明。
  - 问题代码、级别、说明和关联备份项。
  - 不迁移登录令牌、浏览器会话、机器标识和 sandbox secrets 的安全说明。

### C-11 移动硬盘结构端到端演练

## 当前安全边界

- 不迁移登录令牌、浏览器会话、机器标识或 sandbox secrets。
- 默认不覆盖新电脑项目。
- 默认不恢复 Codex 原生状态。
- Codex 原生状态恢复要求适配器版本兼容，并要求 Codex 已安装、已初始化、已登录且完全退出。
- 恢复前先校验备份包；恢复后校验目标文件。
