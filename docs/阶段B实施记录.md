## 已完成

### B-01 正式解决方案

- 创建 `CodexBackup.slnx`。
- 创建 WPF 桌面应用、核心领域库、Windows 基础设施库和自动测试项目。
- 启用可空分析、确定性构建和警告视为错误。
- Release 构建通过，0 警告、0 错误。

### B-02 更正后的项目发现模型

- 定义 `CodexSessionPath`、`GitRepository`、`ProjectMarker` 和 `UserAdded` 发现来源，允许同一项目同时保留多个来源。
- 实现 `CodexSessionPathReader`，只读取会话前 32 行中的 `payload.cwd`，找到后立即停止。
- 不读取对话正文，不解析项目文件内容。
- 缺失路径和无法读取的会话转化为可观察警告，不使整个扫描失败。

### B-03 备份计划模型

- 建立项目、会话、状态、配置、技能、插件、生成资源和环境清单类型。
- 凭据和临时数据即使被误选也强制排除。
- `UnknownReviewRequired` 未完成人工决定时必须阻止导出。
- 备份计划保留项目发现来源，便于界面区分 Codex 记录命中项和补漏项。

### B-04 manifest 核心模型

- 建立版本化 `BackupManifest` 和 `BackupManifestEntry`。
- 实现相对路径、目录穿越、重复路径、文件大小和 SHA-256 格式校验。
- 实现带驼峰属性名和字符串枚举的 JSON 序列化。

### B-05 界面骨架

- 建立“从旧电脑导出”、“在新电脑恢复”和“检查现有备份”三入口首页。
- “开始扫描”已连接正式 C# 发现引擎，恢复和备份校验按钮仍保持禁用，不向用户伪装成可用功能。

### B-06 项目根目录解析与轻量补漏

- Codex `cwd` 优先向上解析 Git 根目录；无 Git 时使用最近项目标志目录。
- Codex 明确记录但没有 Git/项目标志的目录不会被丢弃，而是标记为“需确认”并默认不选。
- 补漏扫描只读取目录和文件名，跳过系统目录、`.codex`、依赖缓存和重解析点。
- 同一项目的 Codex、Git、项目标志和手动来源会按根目录去重合并。
- 界面已可展示项目、发现来源、Codex 引用数、确认状态和路径。

### B-07 真实环境只读验证

- 可从 Codex 会话文件成功读取 `cwd` 并归并唯一工作路径。
- 合并 Codex 主来源与项目标志补漏后可得到候选项目。
- 候选项目会区分 Codex 记录相关、仅由补漏发现和 Git 仓库。
- 仅有 Codex 路径但无项目标志的目录会标记为需确认，默认不选。
- 扫描未生成包含私有路径的持久报告。
- 访问受限目录会转化为警告，不中断整体扫描。

### B-08 Git 状态与项目体量只读分析

- 使用 `GIT_OPTIONAL_LOCKS=0` 读取分支、工作区变更、未跟踪文件和远程仓库有无，并设置单仓库命令超时。
- 逐目录读取文件名、大小和属性，不打开项目文件内容；跳过重解析点，避免链接循环。
- 统计文件数量、总大小、大文件数量和潜在敏感文件名，仅向界面显示数量提示。
- 单个 Git 仓库或目录读取失败会转化为警告，不中断其他项目分析。

### B-09 正式备份计划与重叠保护

- 用户勾选后的项目可转换为正式 `BackupPlan`，保留发现来源和预计体量。
- 仅有 Codex 路径、没有 Git 或项目标志的目录默认不选；用户明确勾选后视为已确认。
- 未勾选的待确认项目不会阻塞其他已选项目。
- 已选项目存在父子路径重叠时阻止导出，并在界面中指出“哪个项目包含哪个项目”。
- 当前按钮只生成计划，不复制、压缩或修改任何项目文件。

### B-10 真实环境复验

- 候选项目可完成文件元数据盘点。
- Git 项目状态读取失败会被记录为警告，不中断其他项目。
- 默认选择生成的计划可识别路径重叠并阻止继续，证明保护规则已生效。
- 聚合探针不输出项目路径；详细路径只在本机桌面界面中供用户确认。

### B-11 可验证项目导出引擎

- 导出前重新盘点已选项目，得到精确文件数、字节数和最大文件大小，并执行目标空间与 FAT32 大文件检查。
- 目标目录不得与任何源项目互相包含，防止递归复制或把备份写回源项目。
- 文件使用 1 MiB 缓冲区流式复制，不把大文件整体载入内存；保留文件修改时间和空目录。
- 复制阶段计算源 SHA-256；全部复制结束后，再从目标介质完整重读并比对 SHA-256。
- 源文件在复制期间发生大小或修改时间变化时，导出失败并标记为可重试。
- 完成包包含 `manifest.json`、`package-index.json`、JSON/HTML 报告和分类 payload。

### B-12 未完成包、取消与安全提交

- 写入期间目录使用 `.incomplete` 后缀并包含 `INCOMPLETE.json`。
- 空间不足发生在写入前，不创建半成品；复制失败、目标读回失败或用户取消时保留可定位的不完整包和错误报告。
- 只有全部目标文件通过校验后，才删除未完成标记并在同一卷内将目录转为正式完成包。
- 独立只读校验器可重新验证 manifest、package index、文件大小和 SHA-256，并拒绝不完整包。
- 包路径规则拒绝目录穿越、NTFS 备用数据流、Windows 保留名称、重复路径和未映射条目。

### B-13 桌面项目导出测试版

- 备份计划无阻止问题时，可选择目标文件夹并导出已选项目。
- B-13 阶段的测试版在写入前显示项目数量、预计大小和疑似敏感文件风险；当时尚未包含 Codex 数据，后续已由 B-14 至 B-21 补齐。
- 导出中显示阶段、文件数、字节数、速度和预计剩余时间，并提供安全取消按钮。
- 项目勾选发生变化后会立即使旧计划失效，必须重新生成计划，避免界面选择与实际导出不一致。

### B-14 版本化 Codex 本地存储适配器

- 新增 `windows-local-1.0` 适配器，只读取 `.codex` 顶层名称、类型、文件数量和大小，不读取凭据内容或对话正文。
- 会话、规则、技能、插件、生成资源、配置和已知状态库进入原始快照候选。
- `models_cache.json` 与 `vendor_imports` 仅记录清单；未知顶层项默认不选并要求用户确认。
- Codex 具体目录映射仅视为当前适配器观测，不宣称为永久公开格式。

### B-15 凭据与临时数据强制排除

- `auth.json*`、`.cockpit_codex_auth.json*`、sandbox secrets、安装标识、机器绑定标识和 Chrome native host 状态归为凭据并强制排除。
- 缓存、sandbox 运行时、浏览器/电脑控制运行时、临时目录和进程管理数据归为可重建临时数据并强制排除。
- 配置、会话和插件可能含私人信息，界面会持续提示目标介质必须可信。
- 端到端测试确认完成包包含会话、规则和配置，但 manifest 中不存在凭据与缓存文件。

### B-16 Codex 运行状态与数据库一致性保护

- 扫描 Codex 进程数量，并以独占只读方式检查顶层状态库和专用 `sqlite` 目录中的数据库占用。
- 记录 SQLite WAL/SHM/journal 边车文件数量；快照时保留数据库目录层次和边车文件。
- 只要 Codex 进程仍在运行或数据库仍被锁定，包含原生 Codex 数据的计划就加入阻止问题。
- 开始写入前再次检查运行状态，防止用户生成计划后重新启动 Codex。

### B-17 桌面 Codex 数据清单

- 桌面扫描结果增加“Codex 数据（凭据强制排除）”标签页，展示类别、策略、体量、风险和分类原因。
- 凭据、临时数据和仅清单项不能被误勾选；未知项默认不选。
- 项目与 Codex 数据合并进入同一备份计划、manifest、package index 和完整校验流程。
- B-17 阶段先保存原始会话文件；通用对话 JSON/Markdown 随后已由 B-19 至 B-21 实现并接入校验。

### B-18 聚合复验

- 可发现 Codex 顶层项并归类为默认包含、凭据排除、临时排除、仅清单和未知待确认。
- 聚合报告不包含顶层名称、路径、凭据或对话正文。
- 当检测到 Codex 进程、锁定数据库或数据库边车文件时，原生快照会被正确阻止。

### B-19 通用对话结构指纹与容错解析

- 对会话只统计事件类型与字段名称，不输出消息正文；确认可见消息主要来自 `event_msg/user_message` 和 `event_msg/agent_message`。
- `developer` 指令、工具调用、工具输出、token 统计、内部 reasoning、上下文压缩数据和其他运行事件不进入通用对话。
- 每个会话记录源 CLI 版本和基于事件字段集合计算的 SHA-256 结构指纹，不假设 Codex JSONL 是永久公开格式。
- 单行 JSON 损坏会记录行号并继续解析其他消息；缺少会话 ID 时生成稳定替代 ID。
- 会话回滚事件计数写入 JSON/Markdown，并提示通用副本按原始事件出现顺序保留可见消息。

### B-20 脱敏 JSON 与 Markdown

- 每个会话生成独立 JSON 与 Markdown，包含时间、用户/Codex 角色、原工作目录、项目关联、消息数量和转换警告。
- 自动替换常见 OpenAI/GitHub/AWS token、Bearer token、密码/令牌/密钥赋值和 PEM 私钥块。
- Markdown 转义消息中的 HTML 标签，避免曾经粘贴的网页脚本被阅读器当作 HTML 执行。
- 用户图片只记录附件数量，不在通用文件中保存 URL 或 data URI；原始附件信息仍保存在原始会话快照中。
- `portable/conversations/index.json` 建立会话到 JSON、Markdown 和已备份项目 ID 的关联。

### B-21 导出与完整校验集成

- 通用对话从备份包内已经复制完成的原始 JSONL 生成，保证派生文件与原始快照属于同一时点。
- 通用文件、索引和原始会话一起进入 manifest，并执行目标介质 SHA-256 重读校验。
- 单个会话解析警告会产生 `PartialSuccess` 完成包；原始会话仍保留，不会因通用转换失败而丢失。
- 独立校验器除哈希外，还检查通用索引版本、路径、会话 ID、消息数量、角色数量、脱敏数量和消息序号。
- 目标空间预估为通用对话额外预留不低于所选原始会话大小的空间。

### B-22 真实会话只读复验

- 对真实会话执行聚合解析，验证解析器可处理当前观测到的结构变体。
- 可统计可见消息数量、用户/Codex 角色数量、解析警告和读取失败数量。
- 可检测并脱敏疑似秘密。
- 可识别结构指纹和包含回滚事件的会话，证明版本容错和回滚提示不可省略。
- 聚合输出未包含消息、标题、会话 ID、文件名或路径。
