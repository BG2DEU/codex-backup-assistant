# 故障代码与日志说明

## 失败时先看哪里

1. 先看软件底部状态栏。
2. 如果已经生成恢复报告，点击“打开恢复报告”。
3. 点击“打开日志目录”，查看当天的 `codex-backup-YYYYMMDD.log`。
4. 如果是备份包问题，查看备份包内的 JSON/HTML 导出报告。

## 日志位置

桌面程序日志默认写入：

```text
%LocalAppData%\CodexBackupAssistant\logs
```

日志只记录操作阶段、路径、结果和故障代码；不记录对话正文，不记录登录令牌。

常见日志事件：

| 事件 | 含义 |
| --- | --- |
| `APP_START` | 程序启动和版本号 |
| `SCAN_START` / `SCAN_SUCCESS` / `SCAN_FAILED` | 扫描开始、成功、失败 |
| `PLAN_READY` / `PLAN_BLOCKED` | 备份计划可导出或被阻止 |
| `EXPORT_START` / `EXPORT_RESULT` / `EXPORT_EXCEPTION` | 导出开始、结果、异常 |
| `VERIFY_START` / `VERIFY_RESULT` / `VERIFY_EXCEPTION` | 备份包校验开始、结果、异常 |
| `RESTORE_START` / `RESTORE_RESULT` / `RESTORE_EXCEPTION` | 恢复开始、结果、异常 |
| `OPEN_RESTORE_REPORT` | 打开恢复报告 |

## 常见故障代码

| 代码 | 常见原因 | 处理方式 |
| --- | --- | --- |
| `CODEX_MUST_BE_STOPPED` | Codex 仍在运行或数据库被占用 | 完全退出 Codex，重新扫描并生成计划 |
| `PLAN_NO_INCLUDED_ITEMS` | 没有任何可导出的项目或 Codex 数据 | 重新扫描并确认勾选了需要备份的项目 |
| `PLAN_REVIEW_REQUIRED` | 有未知 Codex 数据需要人工确认 | 检查 Codex 数据页，确认是否需要勾选未知项 |
| `PLAN_OVERLAPPING_INCLUDED_PATHS` | 已选项目存在父子目录重叠 | 只保留真正需要备份的上层或下层项目 |
| `PACKAGE_MISSING` | 选择的备份包目录不存在 | 重新选择包含 `manifest.json` 的备份包根目录 |
| `PACKAGE_INCOMPLETE` | 备份包未完成或包含 `INCOMPLETE.json` | 回到旧电脑重新导出，不要用未完成包恢复 |
| `PACKAGE_METADATA_MISSING` | 缺少 `manifest.json` 或 `package-index.json` | 选择备份包根目录，不要选择 `payload`、`portable` 或 `tools` 子目录 |
| `TARGET_LENGTH_MISMATCH` | 备份包文件大小与清单不一致 | 备份包可能损坏，重新从旧电脑导出或重新复制移动硬盘 |
| `TARGET_HASH_MISMATCH` | 备份包文件 SHA-256 不一致 | 备份包可能损坏，重新导出或更换移动硬盘 |
| `RESTORE_CODEX_NOT_INITIALIZED` | 新电脑 Codex 目录不存在 | 在新电脑启动并登录 Codex 一次，然后退出 Codex 再恢复 |
| `RESTORE_CODEX_STILL_RUNNING` | 恢复 Codex 原生状态时 Codex 仍在运行 | 完全退出 Codex；普通恢复建议保持“不恢复原生状态” |
| `RESTORE_DUPLICATE_TARGET` | 多个备份项映射到同一恢复位置 | 更换项目恢复目录，或减少要恢复的项目 |
| `RESTORE_NO_READY_ITEMS` | 没有任何可恢复项 | 确认备份包有效，并确认恢复选项没有全部关闭 |
| `RESTORE_CANCELLED` | 用户取消恢复 | 如需继续，重新执行恢复；已写入内容会按回滚记录处理 |
| `RESTORE_ROLLBACK_FAILED` | 回滚时某个文件无法删除或恢复 | 查看日志中的具体路径，手动检查文件是否被占用 |
| `RESTORE_UNEXPECTED_FAILURE` | 未预期异常 | 查看日志和恢复报告，保留备份包，不要删除回滚目录 |

## 需要保留给排查的材料

如果真实换机测试失败，请保留：

- 备份包根目录。
- 备份包内 `manifest.json` 和 `package-index.json`。
- 备份包内导出报告。
- 新电脑恢复目录中的 `Codex换机恢复资料`。
- 恢复报告 HTML/JSON。
- `%LocalAppData%\CodexBackupAssistant\logs` 中当天日志。

## 判断问题类型

- 发生在“开始扫描”阶段：通常是源项目、Codex 目录或权限问题。
- 发生在“导出备份包”阶段：通常是移动硬盘空间、文件占用、路径重叠或源文件变化。
- 发生在“校验备份包”阶段：通常是选错目录、备份包未完成或移动硬盘损坏。
- 发生在“选择备份并恢复”阶段：通常是目标目录权限、Codex 未退出、备份包不兼容或目标文件被占用。
